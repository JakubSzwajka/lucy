# Session Routes

## Endpoints

- `GET /api/sessions`
- `POST /api/sessions`
- `GET /api/sessions/[id]`
- `PATCH /api/sessions/[id]`
- `DELETE /api/sessions/[id]`
- `POST /api/sessions/[id]/chat`
- `GET /api/sessions/[id]/plans`

## Purpose

User-facing conversation/session lifecycle and turn execution entrypoint.
